# 機能一覧と役割

対象バージョン: 0.5.0

## このガイドについて

Kinmu-OSは、従業員100名以下の組織を対象にしたセルフホスト型の労務・勤怠管理ソフトです。このガイドは所有者、労務管理者、従業員が現在利用できる機能と操作を説明します。

## 役割と権限

- 所有者: 初期設定、利用者、従業員、部署、勤務ルール、勤務カレンダー、休暇、勤怠、残業申請設定・審査、レポート、監査ログを管理できます。
- 労務管理者: 所有者と同じ日常管理機能を利用できます。
- 従業員: 自分の打刻、勤務実績、勤怠修正申請、休暇残高・申請・取消、残業・休日出勤申請、通知、プロフィールを利用できます。

## 機能一覧

- 組織の初期設定とログイン利用者の管理
- 部署と従業員台帳、利用者との紐付け
- 所定開始・終了時刻、休憩、所定労働時間の勤務ルール
- 適用日付き勤務曜日、会社休日、臨時勤務日、従業員別例外の勤務カレンダー
- 休暇種別、半日単位の付与・調整・失効と追記型残高台帳
- 全日・半日休暇の申請・取消と単段階審査、理由付き欠勤確定
- 出勤、退勤、休憩開始、休憩終了の打刻
- 実労働時間と残業時間の集計
- 勤怠修正の申請、取消、承認、却下
- 適用日付き残業申請ポリシー、残業・休日出勤申請、単段階審査、実績差異
- 申請提出・取消・審査結果のアプリ内通知と未読管理
- 未退勤、審査待ち、未解決、日区分競合を確認する月次締め、理由付き再開、再締め
- 従業員CSV、勤怠CSV、休暇台帳CSVの出力
- 重要操作の監査ログ

詳しい導入手順は[初期設定と従業員管理](admin-setup.md)、[勤務カレンダーの設定](work-calendar.md)、[休暇種別・付与・残高管理](leave-management.md)、[残業申請設定・審査・差異](overtime-management.md)、日々の操作は[打刻・勤務実績・プロフィール](employee-attendance.md)、[休暇申請・審査・欠勤確定](leave-requests.md)、[残業・休日出勤申請](overtime-requests.md)を参照してください。

## 現行版の制限

時間単位休暇、自動法令判定、多段承認、管理者による代理申請には対応していません。アプリ内通知は残業・休日出勤申請のイベントが対象です。集計と申請承認は法令や就業規則への適合を判定するものではありません。
