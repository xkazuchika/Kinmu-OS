# 残業申請設定・審査・差異

対象バージョン: 0.5.0

## ポリシーを設定する

所有者または労務管理者が「残業申請設定」を開き、入力単位、事前申請、実績差異の許容分数、未解決差異による締め阻害、適用開始日をドラフト保存します。既存組織には未有効の初期ドラフトが用意されるため、内容を確認するまで従来の勤怠動作は変わりません。

「影響を確認して有効化」で対象従業員、既存申請、締め済み月を確認します。締め済み月へ効力を持つ設定は有効化できません。必要なら対象月を再開するか、未締め月を適用開始日にします。設定は適用日付きで履歴を保持し、勤務日ごとに有効な最新設定を使います。

## 単段階で審査する

1. 「残業審査」で状態、区分を絞り込みます。
2. 申請を選び、理由、申請時間、勤務予定、カレンダー根拠、現在の打刻、同日の申請を比較します。
3. 承認するか、具体的な理由を入力して却下します。

申請者本人は自分の申請を承認できません。承認時に在籍、月次締め、ポリシー、勤務カレンダー、重複、申請バージョンを再検査します。競合した場合は再読み込みして最新状態を確認してください。

## 差異と月次締め

勤怠一覧では申請状態と実績差異状態を別々に絞り込めます。所定勤務日は残業分、休日は実労働分を承認済み申請の合計と比較します。勤怠修正や打刻更新後は未締め月の現在値から再計算されます。

審査待ち申請は常に月次締めを阻害します。「未解決差異がある月を締めない」を有効にした場合、申請超過、実績なし、未申請の実績も阻害します。申請未満は締めを阻害しません。締め時には申請と差異をリビジョンへ固定し、締め済み一覧とCSVは固定値を使います。詳しくは[月次勤怠の締め・再開](monthly-closing.md)を参照してください。

## 現行版の制限

時間単位休暇、自動法令判定、多段承認、管理者による代理申請には対応していません。申請・承認は36協定、法定休日労働、割増賃金などの法令や就業規則への適合を判定・保証するものではありません。
