# トラブル対処

対象バージョン: 0.5.0

## ログインと権限

- ログインできない: メールアドレス、12文字以上で設定したパスワード、利用者が有効かを管理者と確認します。
- 管理画面が見えない: 従業員には管理画面を表示しません。所有者または労務管理者の役割を確認します。
- 勤務実績が見えない: 管理者が従業員台帳とログイン利用者を紐付けているか確認します。

## 打刻と申請

- 打刻できない: 出勤、休憩開始、休憩終了、退勤の順序と、当日の現在状態を確認します。
- 未退勤になる: その日の退勤打刻がないためです。勤務実績から修正を申請します。
- 修正が競合する: 審査待ち申請を取り消し、最新の勤務実績を開いて再申請します。
- 休暇を申請できない: 勤務カレンダーの有効化、申請可能な休暇種別、残高、対象日の締め状態を管理者と確認します。
- 休暇を承認できない: 自己承認、全日休暇と打刻の重複、別の休暇、残高更新、月次締めの競合を確認します。
- 残業申請を送信できない: 入力単位、事前申請、24時間上限、予定休憩、同じ時間帯の重複、勤務カレンダー、月次締めを確認します。
- 残業申請を審査できない: 自己承認、古い画面のバージョン、在籍、ポリシー・カレンダー変更、重複、月次締めを確認して再読み込みします。
- 実績差異が解消しない: 承認申請と打刻・勤怠修正を照合します。申請分は実績へ加算されません。所定日は残業分、休日は実労働分を比較します。
- 月を締められない: 審査待ち休暇・残業、設定された残業差異、未解決勤務日、日区分競合を「勤怠」「休暇審査」「残業審査」から解消します。
- 通知の対象を開けない: 通知後の役割・対象状態変更により導線が無効になっています。通知は既読にして、現在表示できる申請一覧から確認します。

## CSV

対象月、ダウンロード権限、ブラウザのダウンロード設定を確認します。勤怠CSVの値が想定と違う場合は、承認済み修正と適用中の勤務ルールを確認します。

## セルフホスト環境

1. `/api/health` が成功するか確認します。
2. アプリ、migrator、databaseのコンテナログを確認します。
3. `APP_URL`、データベース接続、シークレット、TLSリバースプロキシを確認します。
4. 更新前バックアップと、稼働版に対応するソースURLを確認します。

ログや画面を共有するときは、パスワード、Cookie、接続文字列などの秘密情報を除いてください。[機能一覧と役割](overview.md)には現行版の制限も掲載しています。

## 現行版の制限

時間単位休暇、自動法令判定、多段承認、管理者による代理申請には対応していません。アプリ内通知は残業・休日出勤申請のイベントが対象です。集計と申請承認は法令や就業規則への適合を判定するものではありません。
