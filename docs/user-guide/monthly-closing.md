# 月次勤怠の締め・再開

対象バージョン: 0.5.0

## 対象役割と目的

所有者と労務管理者が「勤怠」で対象月を確認し、給与計算などへ渡す勤務実績をリビジョン付きで確定します。締められるのは終了した月だけです。

## 締め前に確認する

1. 「勤怠」で対象月を選び、「表示」を選びます。
2. 月次勤怠カードで従業員数、勤務日数、休暇、欠勤、未退勤、勤怠・休暇・残業の審査待ち、設定された残業差異、未解決、日区分競合、集計未作成を確認します。
3. 未退勤がある場合は対象一覧を確認し、正しい打刻または修正申請で解消します。
4. 審査待ちがある場合は「勤怠申請」「休暇審査」または「残業審査」で承認・却下します。
5. 未解決日は打刻修正または理由付き欠勤確定、日区分競合は勤務カレンダーと休暇を確認して解消します。
6. 阻害件数が0になったら、一覧、休暇・欠勤、残業集計が想定どおりか確認します。

残業時間は勤務ルールとの差分であり、法令、就業規則、36協定への適合を自動判定しません。給与連携前に組織の規程に沿って確認してください。

## 月を締める

1. 「この月を締める」を選びます。
2. 対象月と確定後に変更できなくなることを確認し、「締めて確定」を選びます。
3. 「締め済み」、締めた利用者、日時、リビジョンが表示されたことを確認します。
4. 「レポート」から勤怠CSVを出力し、対象月とリビジョンを含む名前で安全に保管します。

締め済み月では通常打刻、勤怠・休暇・残業の申請・取消・審査、対象期間のポリシー有効化、カレンダー変更、残高変更、欠勤確定はできません。従業員の勤務実績にはカレンダー根拠、日区分、休暇名・単位・対応所定時間、欠勤理由、残業申請と実績差異を含む確定値と締めリビジョンが表示されます。

## 理由付きで再開して再締めする

1. 対象月の「締めを再開」を選びます。
2. 修正が必要になった経緯を5文字以上で入力し、「再開する」を選びます。
3. 必要な打刻または修正申請と審査を行い、締め前検査をもう一度確認します。
4. 「この月を締める」から再締めし、リビジョンが増えたことを確認します。
5. 新しい確定CSVを保管し、給与連携などの利用先へリビジョン変更を伝えます。

過去の締めリビジョン、再開理由、締め・再開・再締めの操作は監査履歴として保持されます。月次運用の前後ではデータベースのバックアップも取得してください。

## 現行版の制限

時間単位休暇、自動法令判定、多段承認、管理者による代理申請には対応していません。アプリ内通知は残業・休日出勤申請のイベントが対象です。集計と申請承認は法令や就業規則への適合を判定するものではありません。
