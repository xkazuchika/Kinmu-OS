# 月次勤怠の締め・再開

対象バージョン: 0.3.0

## 対象役割と目的

所有者と労務管理者が「勤怠」で対象月を確認し、給与計算などへ渡す勤務実績をリビジョン付きで確定します。締められるのは終了した月だけです。

## 締め前に確認する

1. 「勤怠」で対象月を選び、「表示」を選びます。
2. 月次勤怠カードで従業員数、勤務日数、未退勤、審査待ち、集計未作成を確認します。
3. 未退勤がある場合は対象一覧を確認し、正しい打刻または修正申請で解消します。
4. 審査待ちがある場合は「勤怠申請」で承認または却下します。
5. 件数が0になったら、一覧と残業集計が想定どおりか確認します。

残業時間は勤務ルールとの差分であり、法令、就業規則、36協定への適合を自動判定しません。給与連携前に組織の規程に沿って確認してください。

## 月を締める

1. 「この月を締める」を選びます。
2. 対象月と確定後に変更できなくなることを確認し、「締めて確定」を選びます。
3. 「締め済み」、締めた利用者、日時、リビジョンが表示されたことを確認します。
4. 「レポート」から勤怠CSVを出力し、対象月とリビジョンを含む名前で安全に保管します。

締め済み月では通常打刻、修正申請、申請取消、承認、却下はできません。従業員の勤務実績には確定値と締めリビジョンが表示されます。

## 理由付きで再開して再締めする

1. 対象月の「締めを再開」を選びます。
2. 修正が必要になった経緯を5文字以上で入力し、「再開する」を選びます。
3. 必要な打刻または修正申請と審査を行い、締め前検査をもう一度確認します。
4. 「この月を締める」から再締めし、リビジョンが増えたことを確認します。
5. 新しい確定CSVを保管し、給与連携などの利用先へリビジョン変更を伝えます。

過去の締めリビジョン、再開理由、締め・再開・再締めの操作は監査履歴として保持されます。月次運用の前後ではデータベースのバックアップも取得してください。
