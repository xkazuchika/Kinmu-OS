# 勤務カレンダーの設定

対象バージョン: 0.5.0

## 対象役割と設定順序

対象役割は所有者と労務管理者です。「勤務カレンダー」で、曜日パターン、会社の日付例外、必要に応じて従業員別例外の順に設定します。最初のドラフトは有効化するまで勤怠へ影響しません。

## 曜日パターンを有効化する

1. 適用開始日と通常の勤務曜日を指定し、「ドラフトを作成」を選びます。
2. 一覧で勤務曜日と適用開始日を確認し、「影響を確認」を選びます。
3. 対象従業員数と適用開始日を確認し、「この内容で有効化」を選びます。
4. 「勤怠」で勤務日・休日と勤務予定の根拠を確認します。

適用開始日より前へは遡りません。締め済み月へ影響する開始日は拒否されるため、必要なら先に[月次勤怠の締め・再開](monthly-closing.md)に従って月を再開します。

## 会社休日と臨時勤務日

「会社・従業員の日付例外」で対象を会社全体にし、日付、休日または臨時勤務日、名称、理由を入力します。従業員ごとに異なる日だけ対象者を選びます。判定優先順位は従業員別例外、会社例外、曜日パターンです。

無効化時も理由を入力します。締め済み月の日付例外は変更できません。

## 会社休日CSV

1. UTF-8で `date,kind,name,reason` の固定ヘッダーを用意します。
2. `kind` は休日なら `non_workday`、臨時勤務日なら `workday` にします。
3. ファイルを選び「検証する」で追加・変更・拒否件数と行別エラーを確認します。
4. エラーが0件なら「確定して取り込む」を選びます。

同じ内容のCSVは再取り込みできません。1行でも不正な場合は全行を反映しません。

## 現行版の制限

時間単位休暇、自動法令判定、多段承認、管理者による代理申請には対応していません。アプリ内通知は残業・休日出勤申請のイベントが対象です。集計と申請承認は法令や就業規則への適合を判定するものではありません。
