# 休暇種別・付与・残高管理

対象バージョン: 0.5.0

## 対象役割と準備

対象役割は所有者と労務管理者です。「休暇管理」で休暇種別を作り、従業員へ残高を付与します。休暇申請を始める前に[勤務カレンダーの設定](work-calendar.md)を有効化してください。

## 休暇種別を作成する

コード、表示名、有効期間、有給・無給、残高消費、従業員申請可否を指定します。利用開始後のコード、有給区分、残高消費区分は変更できません。制度を変える場合は既存種別を無効化して新しい種別を追加します。無効化しても過去の申請と台帳は残ります。

## 手動で付与・調整する

1. 処理、従業員、休暇種別、付与・調整日、単位、理由を入力します。
2. 付与では必要に応じて有効期限を入力します。
3. 1単位が半日、2単位が1日であることを確認し、「台帳へ記録」を選びます。
4. 対象従業員を選び、利用可能残高、審査待ち予約、次回失効を確認します。

減額調整は単位を負数にします。元の付与や取引は上書きせず、新しい調整取引として保存されます。

## 休暇付与CSV

1. UTF-8で `employeeNumber,leaveTypeCode,units,grantedOn,expiresOn,reason` の固定ヘッダーを用意します。
2. ファイルを選び「検証する」で対象人数、行数、合計単位、行別エラーを確認します。
3. エラーが0件なら「確定して取り込む」を選びます。

同じ内容のCSVは再取り込みできず、1行でも不正な場合は全行を反映しません。

## 台帳と失効予定を確認する

従業員、休暇種別、期間で付与・調整・消化・戻し・失効を検索します。CSV出力も同じ追記型履歴です。承認消化は有効期限が早い付与ロットから割り当てられます。

## 現行版の制限

時間単位休暇、自動法令判定、多段承認、管理者による代理申請には対応していません。アプリ内通知は残業・休日出勤申請のイベントが対象です。集計と申請承認は法令や就業規則への適合を判定するものではありません。
