# 休暇申請・審査・欠勤確定

対象バージョン: 0.5.0

## 従業員が全日・半日休暇を申請する

1. 「休暇」で種別別の利用可能残高、審査待ち予約、次回失効を確認します。
2. 休暇種別、全日または半日、開始日、終了日、申請理由を入力します。半日は1日だけ指定します。
3. 「対象日と残高を確認」を選び、休日として除外された日、勤務予定、必要日数、申請後残高を確認します。
4. 「この内容で申請」を選び、状態が審査待ちになったことを確認します。

審査待ちの自分の申請だけを取り消せます。承認済み・却下済み申請や締め済み月は変更できません。

## 管理者が審査する

1. 「休暇審査」で審査待ち申請を選びます。
2. 対象日、全日・半日、勤務予定、打刻、既存休暇、台帳残高、審査待ち予約、付与ロットを確認します。
3. 承認する場合は確認ダイアログから確定します。残高は有効期限の早い付与から消化されます。
4. 却下する場合は理由を入力して確定します。

自分の申請は自分で承認・却下できません。承認直前に打刻、重複、残高、締め状態が変わった場合は競合として停止するため、最新状態を確認します。

## 未解決日を欠勤へ確定する

管理者は過去の未締め所定勤務日で、打刻も承認済み休暇もない日を理由付き欠勤へ確定できます。欠勤は休暇残高を消費しません。誤りがある場合は月次締め前に勤怠と休暇の状態を確認してください。

## 月次締めとの関係

審査待ち休暇、未解決の所定勤務日、日区分競合は月次締めを止めます。[月次勤怠の締め・再開](monthly-closing.md)の手順で各件数を0にしてから締めます。締め済みリビジョンには休暇名・単位・対応所定時間、カレンダー根拠、欠勤理由が固定保存されます。

## 現行版の制限

時間単位休暇、自動法令判定、多段承認、管理者による代理申請には対応していません。アプリ内通知は残業・休日出勤申請のイベントが対象です。集計と申請承認は法令や就業規則への適合を判定するものではありません。
