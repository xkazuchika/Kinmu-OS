# v0.1リリース候補レビュー

レビュー日: 2026-07-15

## 判定

v0.1の実装範囲とセルフホスト経路はリリース候補として合格。リリースを妨げる既知の不具合はありません。

## 確認結果

- 所有者、労務管理者、従業員の認証・認可と代表画面を確認
- 従業員台帳、部署、CSV取込プレビュー、従業員・勤怠CSV出力を確認
- 出勤、休憩開始、通信エラー表示、休憩終了、退勤、勤務実績を320 CSSピクセル幅で確認
- 残業列、勤務ルール、監査ログ、利用者無効化の確認ダイアログとキーボード操作を確認
- 実行バージョン、AGPL-3.0-only、対応ソースURLの認証済み画面を確認
- 本番Composeの設定検証、空DBへの自動マイグレーション、ヘルスチェックを確認
- UTC日時付きバックアップ、明示確認なしの復元拒否、空DBへの復元、非空DBへの復元拒否を確認
- 移行成功後にアプリを更新するスクリプトを確認
- 型検査、Lint、整形、14ファイル41テスト、本番Dockerビルド、3本のPlaywright E2Eに合格
- `pnpm audit` は既知の脆弱性0件。製品依存ライセンス一覧を確認

## 性能スモーク

ローカルColima/PostgreSQLで、100名・31日・3,100件の月次勤怠を投入して計測しました。

| 経路                   |    実測 |
| ---------------------- | ------: |
| 管理者向け月次勤怠一覧 | 13.1 ms |
| ダッシュボード集計     | 14.5 ms |
| 月次勤怠CSV生成        | 42.6 ms |

各経路のスモーク上限は3,000 msです。継続的な再計測は `performance.integration.test.ts` が担当します。

## リリース時に必ず確定する項目

- `SOURCE_CODE_URL` を稼働イメージと同じタグまたはコミットの取得先へ変更する
- `APP_VERSION` をリリースタグと一致させる
- `APP_URL` をTLSリバースプロキシで公開する正規HTTPS URLへ変更する
- 初期シークレットをランダム値へ変更し、バックアップの外部・暗号化保管先を用意する
- 公開前に名称・ロゴ方針とAGPL運用文面について必要に応じて法務確認を行う

## v0.1の既知の制約

- 休暇、承認、勤怠訂正、通知、フレックス、シフト、深夜勤務、給与・行政手続きは対象外
- TLS証明書、外部バックアップ保管、監視は導入者が用意する
- 対応ソースURLは自動公開されないため、改変版の運営者が取得先を維持する
