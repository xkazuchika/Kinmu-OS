# 勤怠修正申請の運用

## 役割

- 従業員は自分の勤務実績だけを申請・閲覧し、審査待ちの申請だけを取り消せます。
- 所有者と労務管理者は同じ組織の申請を閲覧し、承認または却下できます。
- 申請者本人は、自分の申請を審査できません。

## 審査

管理者は「勤怠申請」で修正前、申請後、追加・変更・削除の差分と理由を確認します。承認すると元の打刻は削除せず置換済みとして保存され、申請後の打刻から勤務日状態、休憩、実労働、残業が再計算されます。退勤を削除した申請を承認した場合、その勤務日は未退勤へ戻り、確定前の集計は表示されません。却下時はコメントが必須です。

申請後に通常打刻が増減した場合は、古い内容を上書きしないよう承認が競合で停止します。従業員が審査待ち申請を取り消し、最新の勤務実績を確認して再申請してください。

## 監査と月次運用

申請、取消、承認、却下、反映について、操作者、対象従業員、勤務日、申請ID、許可された打刻差分が監査ログに残ります。元の打刻と申請内容も履歴として保持されます。

v0.2には月次締め、締め後の再開、承認経路の多段化、代理申請はありません。給与計算へ渡す前に未処理申請と未退勤が0件であることを確認し、その時点の勤怠CSVを保管してください。
