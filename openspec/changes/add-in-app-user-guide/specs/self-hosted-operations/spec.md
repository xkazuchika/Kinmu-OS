## MODIFIED Requirements

### Requirement: 公開リポジトリの運用資料
システムは、GitHub上の公開リポジトリに、AGPL-3.0ライセンス、導入手順、開発手順、コントリビューション手順、脆弱性報告窓口、リリース方針、Kinmu-OSの名称・ロゴ利用ポリシー、現行バージョンの利用者向け機能一覧と役割別操作マニュアルを含めなければならない（MUST）。文書は、TLS終端、シークレット管理、データバックアップの責任が導入者にあること、AGPL-3.0に基づくソース提供義務、利用者マニュアルが対応する製品バージョンを明記しなければならない（MUST）。機能追加または操作変更を行うリリースでは、対応する利用者マニュアルも同じ変更で更新しなければならない（MUST）。

#### Scenario: 新規利用者が公開リポジトリを確認する
- **WHEN** 新規利用者が公開リポジトリのトップレベル文書を開く
- **THEN** 利用者はAGPL-3.0ライセンス、名称・ロゴ利用ポリシー、導入方法、運用上の責任、支援・脆弱性報告先、現行機能と役割別操作マニュアルを確認できる

#### Scenario: 操作を変更するリリースを準備する
- **WHEN** 開発者が既存画面の操作手順または利用できる役割を変更する
- **THEN** リリース対象には対応するMarkdownマニュアルの更新と、記事カタログ・内部リンクの検証結果が含まれる
