# in-app-user-guide

## Purpose

認証済み利用者が、役割に合う機能説明・操作手順・トラブル対処・制限事項をアプリ内と公開リポジトリで一貫して参照するための要件を定義する。

## Requirements

### Requirement: アプリ内から見つけられる利用ガイド
認証済みの利用者は「このソフト」から利用ガイドの目次へ移動できなければならない（MUST）。目次は製品の対象範囲と機能一覧を表示し、利用者の役割に適した記事を優先しつつ、各記事の対象役割を文言で示さなければならない（MUST）。

#### Scenario: 従業員がガイドを開く
- **WHEN** 従業員が「このソフト」で利用ガイドを選択する
- **THEN** システムは打刻、勤務実績、勤怠修正、プロフィールの記事を先に表示し、管理者向け記事には対象役割を明記する

#### Scenario: 労務管理者がガイドを開く
- **WHEN** 労務管理者が利用ガイドの目次を開く
- **THEN** システムは従業員・部署・利用者管理、勤務ルール、勤怠審査、CSV、監査の記事へ到達できる目次を表示する

### Requirement: 現行機能を説明する役割別操作手順
利用ガイドは、初期設定、従業員・部署・利用者管理、勤務ルール、打刻、勤務実績、勤怠修正申請・審査、CSV、監査ログについて、利用できる役割、開始画面、前提条件、操作手順、完了時に確認できる結果を説明しなければならない（MUST）。画面に存在しない操作や権限を案内してはならない。

#### Scenario: 従業員が勤怠修正手順を確認する
- **WHEN** 従業員が勤怠修正の記事を開く
- **THEN** ガイドは勤務実績から打刻を追加・変更・削除して理由を入力する手順、審査待ちの取消、承認後に集計が変わることを説明する

#### Scenario: 管理者が申請審査手順を確認する
- **WHEN** 所有者または労務管理者が勤怠申請の審査記事を開く
- **THEN** ガイドは差分確認、承認確認、却下コメント、競合時に最新記録から再申請する手順を説明する

### Requirement: トラブル対処と制限事項
利用ガイドは、ログインできない、権限のある画面が見えない、従業員情報が利用者に紐付かない、打刻順序が不正、未退勤、勤怠修正の競合、CSV出力、セルフホスト環境の切り分けを案内しなければならない（MUST）。現行バージョンで未対応の月次締め、多段承認、代理申請と、残業集計が法令適合を自動判定しないことを明記しなければならない（MUST）。

#### Scenario: 修正申請の承認が競合する
- **WHEN** 利用者が勤怠修正の競合メッセージに対する対処を確認する
- **THEN** ガイドは審査待ち申請を取り消し、最新の勤務実績を確認して再申請する手順を示す

#### Scenario: 未対応機能を確認する
- **WHEN** 利用者が機能一覧または制限事項を開く
- **THEN** ガイドは月次締め、多段承認、代理申請が現行バージョンで未対応であることを表示する

### Requirement: リポジトリ文書とアプリ表示の一貫性
アプリ内ガイドは公開リポジトリに保存されたMarkdown本文を基準として表示しなければならない（MUST）。システムは記事カタログとMarkdownファイル、内部リンク、必須見出し、対象バージョンの不整合を自動検証できなければならない（MUST）。

#### Scenario: ガイドをstandalone環境で開く
- **WHEN** 導入者が本番用Dockerイメージを起動し、認証済み利用者がガイド記事を開く
- **THEN** システムは外部ネットワークへ接続せず、配布物に同梱されたMarkdownと同じ記事を表示する

#### Scenario: 存在しない記事を開く
- **WHEN** 利用者がカタログに存在しないガイドslugへ移動する
- **THEN** システムは空白や例外詳細ではなく404を表示し、既知の記事本文を露出しない

### Requirement: レスポンシブでアクセシブルな記事閲覧
利用者は幅320 CSSピクセル以上の画面で横方向スクロールなしに目次と記事を閲覧できなければならない（MUST）。ガイドは階層的な見出し、現在地、ガイド一覧へ戻る導線、キーボードで操作できるリンク、色に依存しない対象役割表示を提供しなければならない（MUST）。

#### Scenario: モバイル幅で記事を読む
- **WHEN** 利用者が幅320 CSSピクセルの画面で長い操作記事を開く
- **THEN** システムは本文、コード、一覧、前後記事の導線を画面幅内に折り返し、横方向スクロールを要求しない

#### Scenario: キーボードで記事間を移動する
- **WHEN** 利用者がキーボードだけで目次から記事を開き、一覧へ戻る
- **THEN** フォーカス可能なリンクと意味のある見出し順序により操作を完了できる
